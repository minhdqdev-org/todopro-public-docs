# Tasks

A task is the smallest unit of work in TodoPro. It can be as simple as a line of text you type into Quick Add, or as detailed as a dated, prioritized, labelled item with subtasks, reminders and attachments.

Tasks live in a project. A task you create without naming a project goes to your **Inbox**, so you never have to decide where something belongs at capture time.

Everything on this page works the same on the web app, the mobile app and the CLI. Your tasks sync between them in real time.

## Anatomy of a task

| Field | What it does |
|---|---|
| **Title** | The one-line name of the task. Required. |
| **Description** | Longer free-form notes attached to the task. Optional. |
| **Project** | The project the task belongs to. Defaults to Inbox. |
| **Section** | A group inside the project, if the project uses sections. |
| **Priority** | P1 (highest) to P4 (lowest). New tasks default to **P4**. |
| **Due date** | When you plan to work on the task. Can include a time. |
| **Deadline** | A separate hard "must finish by" date. Pro feature. |
| **Labels** | Personal coloured tags you can attach to any task. |
| **Reminders** | Notifications before, at, or independent of the due date. |
| **Recurrence** | A repeating rule such as `every monday`. |
| **Subtasks** | Child tasks nested under a parent task. |
| **Dependencies** | Links marking that one task blocks or is blocked by another. |
| **Comments** | Discussion on the task, useful in shared projects. |
| **Attachments** | Files uploaded to the task. |
| **Assignee** | The collaborator responsible, in shared projects. |
| **Location** | A place that can trigger the task. Pro feature. |
| **Urgent / important** | Eisenhower classification used by the matrix view. |

!!! tip
    You rarely need to fill these in by hand. Typing one line into Quick Add — `Email Sam about the deck tomorrow 9am #Work @email p1` — sets the title, due date, project, label and priority in one go. See [Quick Add syntax](../getting-started/quick-add-syntax.md).

## What you can do with a task

Tasks can be completed and reopened, rescheduled, moved between projects and sections, paused and resumed, duplicated from a template, shared via a public link, and deleted. Recurring tasks can also have a single occurrence skipped. Every task keeps an activity history of what changed and when.
